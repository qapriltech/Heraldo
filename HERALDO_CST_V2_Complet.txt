1.2 Architecture multi-tenant étendue pour M10
M1-M9 : architecture schema-per-tenant standard. Un ministère = un tenant = un schéma PostgreSQL. M10 AGENCE : architecture sous-tenant imbriquée. Une agence = un tenant HERALDO standard + N sous-tenants clients. Chaque client est isolé dans un schéma prefixé agency_{id}_client_{id}. L'isolation est garantie au niveau base de données — aucune requête inter-clients sans privilège explicite.

4. Sécurité
4.1 Authentification
	•	JWT Access token : 15 min — signé RS256 avec clé privée par tenant.
	•	Refresh token : 7 jours — stocké en base, révocable, rotation à chaque usage.
	•	2FA TOTP obligatoire pour rôles Admin, SG, AGENCY_ADMIN.
	•	SSO SAML 2.0 optionnel pour institutions avec Active Directory.

4.2 Sécurité tokens OAuth M10
	•	Chiffrement AES-256-GCM de chaque token — clé dérivée par couple agency_id+client_id.
	•	Clé maître stockée dans HashiCorp Vault — jamais en base.
	•	Rafraîchissement automatique si expiration < 1 heure (Bull background job).
	•	Notification immédiate si rafraîchissement impossible + désactivation du compte social.

4.3 Protection APIs
	•	Rate limiting : 100 req/min par IP, 1000/min par token authentifié.
	•	CORS whitelist stricte par tenant.
	•	CSRF tokens sur toutes les mutations.
	•	Input validation class-validator sur tous les DTOs.
	•	SQL injection : ORM + requêtes paramétrées exclusivement.
	•	XSS : DOMPurify + Content Security Policy headers.

4.4 Sécurité mobile M10 A9
	•	Biométrie Face ID/Touch ID + fallback PIN 6 chiffres.
	•	JWT dans MMKV chiffré — jamais AsyncStorage.
	•	Session expire après 30 min d'inactivité.
	•	Certificate pinning — protection MITM.
	•	Détection jailbreak/root — désactivation automatique.
	•	Screenshot protection sur écrans sensibles (INBOX, CRM, détail client).

7. Spécifications techniques M10 — Points critiques
7.1 Génération PDF rapports AGENCE
	•	Puppeteer headless — rendu HTML/Handlebars → PDF haute qualité.
	•	Chart.js rendu côté serveur — graphiques vectoriels intégrés.
	•	Watermark supprimé en plan PRO et AGENCE.
	•	Stockage dans MinIO — URL signée 7 jours envoyée au client.
	•	Génération asynchrone via Bull job — notification push à la fin.

7.2 Mobile React Native — Architecture
	•	Expo SDK 50+ — OTA updates sans publication App Store.
	•	React Query : cache offline via MMKV — données disponibles sans réseau.
	•	Annuaire, agenda 14 jours, derniers communiqués synchronisés en cache local.
	•	Actions offline en file d'attente — envoyées automatiquement au retour du réseau.
	•	Transcription vocale via Expo Audio + OpenAI Whisper API.
	•	QR code scan ACCREDIT via Expo Camera.

7.3 OAuth réseaux sociaux — Sécurité
	•	AES-256-GCM chiffrement — clé dérivée par couple agency+client.
	•	HashiCorp Vault pour la clé maître — jamais stockée en base.
	•	Rafraîchissement automatique Bull background job à expiration - 1h.
	•	Rate limits par réseau respectés via queue avec throttling Bull.
