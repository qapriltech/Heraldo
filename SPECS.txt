I. ARCHITECTURE GÉNÉRALE ET STACK TECHNIQUE

1.1  Vue d'ensemble de l'architecture
HÉRALDO est construit sur une architecture 3-tiers (Présentation / Logique métier / Données) avec séparation stricte des responsabilités. L'API REST NestJS constitue le cœur de la logique métier.   mfa_enabled      BOOLEAN DEFAULT FALSE,

-- Trigger: déclenche paiement Mobile Money automatique après validation
CREATE OR REPLACE FUNCTION trigger_fcm_payment() RETURNS TRIGGER AS $$
BEGIN
  IF NEW.status = 'VALIDATED' AND OLD.status = 'PENDING' THEN
    PERFORM pg_notify('fcm_payment', row_to_json(NEW)::text);
  END IF;
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

VI. MODULE WEBRTC — SALLES AGORA AUDIO/VIDÉO

6.1  Architecture WebRTC — SFU mediasoup
HÉRALDO utilise une architecture SFU (Selective Forwarding Unit) via mediasoup. Contrairement à une architecture mesh (P2P), le SFU permet à chaque participant de n'envoyer son flux qu'une seule fois vers le serveur, qui le redistribue de manière sélective. Cela permet de supporter 500+ participants sans dégradation des performances.

ARCHITECTURE SFU : Participant A → [Flux audio/vidéo] → Serveur mediasoup → [Sélection] → Participant B, C, D... Chaque participant envoie 1 flux vers le serveur. Le serveur envoie N flux reçus. Avantage : le réseau du participant n'est pas surchargé. Essentiel pour la CI et l'UEMOA.

7.2  Webhooks Mobile Money — Sécurité HMAC-SHA256
// Guard de vérification webhook — commun à tous les opérateurs
@Injectable()
export class WebhookGuard implements CanActivate {
  canActivate(context: ExecutionContext): boolean {
    const req = context.switchToHttp().getRequest();
    const operator = req.params.operator;  // wave | orange | mtn | moov
    const signature = req.headers[`x-${operator}-signature`];
    const secret = this.configService.get(`WEBHOOK_SECRET_${operator.toUpperCase()}`);
    const expectedSig = createHmac("sha256", secret)
      .update(JSON.stringify(req.body)).digest("hex");
    if (signature !== expectedSig) {
      this.auditLogger.warn("WEBHOOK_INVALID_SIGNATURE", { operator, ip: req.ip });
      throw new UnauthorizedException("Invalid webhook signature");
    }
    return true;
  }
}

8.2  Chiffrement des données sensibles
// Chiffrement AES-256-GCM pour données journalistes
import { createCipheriv, createDecipheriv, randomBytes } from "crypto";

const ENCRYPTION_KEY = Buffer.from(process.env.ENCRYPTION_KEY, "hex"); // 32 bytes

export function encrypt(plaintext: string): string {
  const iv = randomBytes(16);
  const cipher = createCipheriv("aes-256-gcm", ENCRYPTION_KEY, iv);
  const encrypted = Buffer.concat([cipher.update(plaintext, "utf8"), cipher.final()]);
  const tag = cipher.getAuthTag();
  return `${iv.toString("hex")}:${tag.toString("hex")}:${encrypted.toString("hex")}`;
}

11.3  Versioning API
Toutes les routes sont préfixées /v1/. Lors de changements breaking, /v2/ sera introduit avec période de support parallèle de 6 mois. Les applications mobiles et desktop doivent inclure le header X-Client-Version pour permettre la détection de versions obsolètes et le blocage des versions expirées (> 90 jours).
